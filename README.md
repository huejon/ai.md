# agent-runtime Prompt Operations

This repository is an owned prompt-engineering operations workspace for agent-runtime. It stores high-signal prompts, agent specs, skills, harnesses, checklists, knowledge, and local runner project configuration used to improve the local agent system.

## Operating Model

- coordinating process is the orchestrator: it decides what daily improvement loop should run and routes work.
- local runner is the programming worker: it makes local engineering edits, runs checks, and reports evidence.
- Repository artifacts and available specialist agents handle prompt architecture, D.A.R.T.E. creation, review, research, and post-task review.
- Artifacts are written in English; conversation with the user is in Portuguese.
- local automation may commit and push routine validated repository maintenance directly to `main`; no delete, deploy, publish, or production mutation happens without explicit approval.

## Long-Horizon Renewal Harness

## Usage

1. Read `AGENTS.md`, `README.md`, and `knowledge/INDEX.md`.
2. For recurring automation, follow `cron-harness/README.md`, continue `work-ledgers/repo-renewal-long/`, and update the durable handoff.
3. For new prompts or skills, use D.A.R.T.E.: Discovery, Architecture, Redaction, Test, Enhance.
4. Keep local runner configuration local-only unless a task explicitly asks for a neutral tracked artifact.
5. When cleanup is uncertain, document it in `cron-harness/cleanup-plan.md` or the active work ledger; delete only when explicitly authorized and evidence-backed.
